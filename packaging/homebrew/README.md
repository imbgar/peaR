# Homebrew distribution

peaR ships as a Homebrew **cask** (a macOS `.app`), updated automatically on every release
by [`.github/workflows/homebrew.yml`](../../.github/workflows/homebrew.yml).

Once set up, users install with:

```bash
brew install --cask imbgar/tap/peaR
```

## One-time setup

1. **Create the tap repo** (public): `imbgar/homebrew-tap`. Homebrew requires the
   `homebrew-` prefix; users reference it as `imbgar/tap`.

2. **Create a PAT** with `repo` scope (or a fine-grained token limited to `homebrew-tap`,
   Contents: read/write).

3. **Add it as a secret** on the `imbgar/pear` repo named **`TAP_GITHUB_TOKEN`**
   (Settings → Secrets and variables → Actions).

That's it. When a release publishes, the workflow finds the release `.dmg`, computes its
SHA-256, and writes/commits `Casks/peaR.rb` in the tap pointing at it. You can also run it
manually via **Actions → Homebrew cask → Run workflow** with a tag.

## Notes
- The cask is `cask "pear"` (lowercase token) named **peaR** — `brew install --cask imbgar/tap/peaR` resolves it.
- Builds are **unsigned**; first launch needs right-click → Open (or `xattr -dr com.apple.quarantine`).
  Signing/notarization can be added later via Apple Developer creds in `release.yml`.
- [`peaR.rb`](./peaR.rb) here is a static reference copy of the cask shape; the live one
  lives in the tap and is regenerated by CI.
